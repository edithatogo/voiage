The lifecourse integration provides a governed boundary for importing and
exporting HEOML-compatible run bundles. It is an integration surface, not a
replacement for the stable numerical kernel.

## Contract and provenance

Use the versioned schemas and illustrative fixtures under the repository's
integration contracts. Preserve the manifest identifier, schema fingerprints,
source checksums, and transformation history with every migrated bundle.

The integration remains subject to source licensing, provenance, compatibility,
and migration evidence. Do not treat an illustrative fixture as evidence from a
real-world source.

See the [data structures](/data-structures/), [migration guide](/user-guide/migration-guide/),
and [developer architecture guide](/developer-guide/architecture/) for the
supported boundaries.
