# lifecourse v1 Illustrative Fixtures

This directory holds non-normative fixtures that document the shared
`lifecourse`/`voiage` contract before the direct adapter path is finalized.

The current illustrative payload shows how a common result envelope should
preserve method settings, scaling metadata, diagnostics, package version
information, and HEOML provenance for EVPI, EVPPI, EVSI, and ENBS families.

It also mirrors the compatibility block used by the normative fixture manifest:
`voiage` `0.2.0`, `lifecourse` profile `v1`, and HEOML profile `0.1`.
That keeps the illustrative contract aligned with the exchange path that
contributors are expected to validate.
