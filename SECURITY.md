# Security Policy

## Supported Versions

Security fixes are applied to the current `main` line and any tagged release
line that is still under active maintenance.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Use the repository's [private security advisory
flow](https://github.com/edithatogo/voiage/security/advisories/new). If that
flow is unavailable, contact the maintainers through the repository security
channel.

When reporting, include:

- a short description of the issue
- the affected version(s)
- the input or command that triggers the problem
- any proof-of-concept details needed to reproduce the behavior

## What Happens Next

We will acknowledge receipt, triage the report privately, and coordinate a fix
or mitigation before any public disclosure.
