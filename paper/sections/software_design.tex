\section{Decision-problem architecture}

\subsection{Stable and frontier method families}

\texttt{voiage} separates methods by evidentiary maturity rather than presenting
every callable function as equally established. Stable methods have versioned
behavior and documented compatibility commitments. Fixture-backed methods have explicit
schemas, examples, and deterministic expected results, but still require
broader methodological and empirical evidence. Experimental capabilities can
change and are isolated from the stable API\@.

\subsection{Semantics for reducing category errors}

A \texttt{ValueArray} records values with named sample and strategy dimensions
and can additionally retain perspective labels. Parameter sets, trial designs,
decision options, populations, and result envelopes provide the corresponding
analysis context. This reduces the risk of a matrix silently changing meaning as it
moves between an EVPI calculation, subgroup analysis, or perspective
comparison. Labeled-array behavior follows the scientific Python convention
represented by \texttt{xarray}~\cite{hoyer2017xarray}.

Each method returns more than a scalar. Results identify the selected strategy,
comparison baseline, method and maturity, diagnostics, and the quantities
needed to explain the value calculation. Population scaling, time horizon,
discounting, implementation assumptions, subgroup and equity weights,
transport weights, data-quality penalties, and stage timing remain visible
where relevant. Invalid dimensions, non-finite values, inconsistent labels,
and impossible rates fail explicitly.

\subsection{Implementation boundary and analytical assurance}

Binding-independent Rust components implement selected stable aggregation
kernels. Python executes user-supplied models and retains orchestration,
validation for broader method paths, labelled data, plots, and reports. For
kernels routed through Rust, this boundary is intended to reduce divergence in
validation, orientation, and tie-breaking behavior. Direct R and Julia support
is currently limited to EVPI, so wider parity is not claimed.

Analytical assurance targets decision properties rather than only code
execution. Tests cover translation and positive-scaling invariance,
non-negativity where required, bounds such as $\mathrm{EVPPI}\leq
\mathrm{EVPI}$, deterministic seeding, rank-deficient regression designs,
degenerate strategies, malformed inputs, and cross-language fixtures.
Property-based, metamorphic, fuzz, mutation, and integration tests complement
example-based checks. These test specified behavior of the implemented
estimators; they cannot validate a user's model structure, evidence, or value
judgments.
