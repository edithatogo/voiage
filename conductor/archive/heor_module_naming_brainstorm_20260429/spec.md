# HEOR Module Naming Brainstorm

## Overview

Capture the short-list naming proposal for future health economics and
outcomes research (HEOR) sibling modules without implying that any of them
should be merged into `voiage`.

## Candidate Names

- `calibrate`
- `evidence`
- `process`
- `report`
- `registry`
- `workflow`
- `quality`
- `engines`
- `heoml`

## Scope

- These names are brainstorming targets only.
- PM4Py stays in the ecosystem brainstorming bucket as a process-mining
  capability, not a current `voiage` module.
- Every future module should declare a CLI surface.
- Every future module should explicitly decide whether an MCP interface is
  useful.

## Acceptance Criteria

- The track records the naming proposal in the repo's Conductor registry.
- The repo docs point to the HEOR naming list and keep the scope separate from
  the current `voiage` implementation.
- Future modules are not described as part of the maintained package by default.
