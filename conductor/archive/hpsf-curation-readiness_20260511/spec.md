# Track Specification: HPSF Curation Readiness

## Overview

HPSF is a software curation and visibility channel, not a package-manager
submission channel. This track documents the metadata and package maturity evidence
needed before requesting curation for `voiage`.

## Functional Requirements

1. Define package identity, licensing, and provenance details needed for HPSF.
2. Define minimum supported platform and dependency disclosure required for listing.
3. Clarify that HPSF does not guarantee install-time delivery; it provides
   discoverability and curation.
4. Define external submission artifact package and maintainer contact points.

## Acceptance Criteria

1. The checklist and contract docs explicitly call out HPSF as curation/visibility.
2. Handoff notes include all required packaging metadata.
3. No incorrect claim of direct publication from this repository.
